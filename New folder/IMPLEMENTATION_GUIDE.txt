1. SYSTEM OVERVIEW
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

RIAMAN FASHION ERP is a sophisticated Excel + VBA-based enterprise resource planning
system designed exclusively for luxury fashion boutiques specializing in high-end
wedding dresses, evening gowns, couture pieces, rentals, tailoring services, and retail.

KEY FEATURES:
✓ Professional boutique management system
✓ POS (Point of Sale) with barcode scanning
✓ Advanced inventory management (movement-based)
✓ Rental management with availability tracking
✓ CRM with VIP customer identification
✓ Complete accounting and financial reporting
✓ Role-based access control (4 user roles)
✓ Audit logging and compliance tracking
✓ Dashboard KPIs and business intelligence
✓ Luxury brand visual identity throughout

TARGET USERS:
• Business owners / Administrators
• Sales staff
• Stock managers
• Accountants

DESIGN PRINCIPLES:

1. SEPARATION OF CONCERNS
   - User interface separate from business logic
   - Business logic separate from data storage
   - All calculations done via VBA, not spreadsheet formulas

2. DATA INTEGRITY
   - Backend sheets completely hidden and protected
   - No manual data entry into backend
   - All changes logged with timestamps and user info
   - Stock levels calculated automatically from movements

3. SECURITY
   - Role-based access control enforced in VBA
   - Password-protected user authentication
   - Sheet protection with restricted access
   - Audit trail of all significant actions

4. SCALABILITY
   - Designed to handle hundreds of products
   - Supports thousands of transactions
   - Rental system with date-based conflict detection
   - Flexible reporting with date range selection

4.4 INVENTORY MANAGEMENT MODULE
────────────────────────────────────────────────────────────────────────────

MOVEMENT-BASED APPROACH:
Stock levels are NEVER manually entered. All inventory changes occur through
documented movements. This ensures complete traceability and eliminates
discrepancies.

PERMITTED MOVEMENTS:
1. Purchase        (+)  New inventory purchased from supplier
2. Sale            (-)  Sold to customer (via POS)
3. Rental Out      (-)  Rented to customer
4. Rental Return   (+)  Rental item returned by customer
5. Damage          (-)  Item damaged and removed from stock
6. Loss            (-)  Item lost or written off

EACH MOVEMENT RECORDS:
• Movement ID          (Unique identifier)
• Product ID           (Which dress)
• Movement Type        (Purchase/Sale/Rental_Out/etc)
• Quantity             (How many units)
• Reference            (Transaction/Rental ID)
• Notes                (Details about movement)
• Timestamp            (When it occurred)
• User                 (Who performed the action)

REAL-TIME STOCK CALCULATION:
Current Stock = SUM of all positive movements - SUM of all negative movements

FEATURES:
✓ Complete movement history for audit
✓ Real-time stock levels always accurate
✓ Low stock alerts (threshold configurable)
✓ Stocktaking and reconciliation via physical count movement
✓ Stock aged reports (when items purchased)
✓ Batch movement recording for efficient operations

STOCK CURRENT SHEET:
Shows live stock levels recalculated whenever movements are recorded

   IF RETURNING:
   a. Click "Select Customer" dropdown
   b. Search by name or phone
   c. Customer history displays (previous purchases, preferences)
   d. Continue to "Add Items"

   IF NEW / WALK-IN:
   a. Click "[Create New Customer]"
   b. Enter name and contact info
   c. Customer ID auto-generated
   d. Continue to "Add Items"

ADDING ITEMS TO SALE:

Method 1: Barcode Scan (Fastest)
a. Ask customer for dress
b. Scan barcode with barcode scanner
c. Item auto-adds to cart with correct price
d. Quantity defaults to 1
e. Repeat for additional items

Method 2: Search by Name
a. Type product name in search box
b. Select from matching results
c. Click to add (or enter quantity)
d. Item adds to cart

Method 3: Visual Selection
a. Click "Browse Products"
b. Product images display
c. Click on dress image
d. Details and price shown
e. Click "Add to Cart"
f. Quantity dialog appears
g. Click "Add"

FOR EACH ITEM, SPECIFY:
• Quantity (how many of this style/size)
• Type: Sale or Rental?
  - If Rental: enter rental days (check availability)
  - If Sale: standard purchase

✓ Line total (quantity × price)
✓ Running subtotal
✓ VAT (5% of subtotal)
✓ Final total

3. Can modify quantities

   ├── Cash

   ├── Stripe (card processing)
   └── Wise (international transfer)

2. If Cash: Take payment, give change
3. If Card: Swipe/tap card, get authorization
4. If Bank Transfer: Provide account details, confirm later
5. If Online/Stripe: Provide payment link to customer

1. Click [COMPLETE PAYMENT]
2. System:
   ├── Generates invoice number (INV-YYYYMMDD-XXXXX)
   ├── Updates customer purchase history
   ├── Updates inventory stock
   ├── Records transaction
   └── Auto-prints receipt
3. Give receipt to customer
4. Thank customer, confirm return visit

4. Confirm rental start and end dates

8. Complete payment

   ├── Creates rental record

    ├── Rental agreement

RENTAL RETURN PROCESS (Stock Manager):

When customer returns rental:
1. Greet customer, check rental ID or phone
2. Inspect dress condition:
   ├── Excellent - No issues, perfect
   ├── Good - Minor staining, easily cleanable
   ├── Fair - Tear, stain, requires repair
   └── Damaged - Major damage, unusable

3. Update Rental Bookings:
   ├── Go to Inventory or Rental tab
   ├── Find rental record
   ├── Click "Process Return"
   ├── Enter return date
   ├── Select condition status
   ├── System calculates late fees if applicable

4. Deposit Handling:
   IF Perfect Condition:
   a. Return full deposit to customer
   b. System records deposit refund
   IF Damaged:
   a. Notify customer of damage assessment
   b. Deduct cleaning/repair costs
   c. Return remaining deposit
   d. Or retain full deposit if not agreed

5. System Auto-Updates:
   ├── Stock movement recorded (Rental_Return +1)
   ├── Dress marked "Available" again
   ├── Late fees recorded if applicable
   ├── Revenue posted to accounting
   └── Audit entry logged

TROUBLESHOOTING & SUPPORT:
─────────────────────────────────────────────────────────────────

Common Issues:

ISSUE: "Macros disabled" on open
SOLUTION:
1. File → Options → Trust Center → Trust Center Settings
2. Macro Settings → Enable all macros
3. Reopen workbook

ISSUE: Login form doesn't appear
SOLUTION:
1. Check _Users_Master sheet exists
2. Verify Workbook_Open event in ThisWorkbook
3. Check VBA for errors (Alt+F11 → Debug → Compile)

ISSUE: Stock levels incorrect
SOLUTION:
1. Check _Stock_Movements for all transactions
2. Run stock reconciliation (manual count)
3. Record adjustment movement to correct

ISSUE: Report numbers don't match
SOLUTION:
1. Verify date range selection
2. Check filter criteria
3. Manually verify sample transactions
4. Review _Accounting_Ledger for all entries
